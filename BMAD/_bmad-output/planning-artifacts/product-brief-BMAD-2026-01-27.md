## Executive Summary

本产品是一款**本地视频播放器 APP**。与常见视频 App 不同，它支持**像音乐 APP 那样锁屏听**：在关屏、通勤、运动等场景下，只播视频里的**音频**，用顺序/循环、通知栏控制等方式听。产品仍以「本地视频播放」为主，只是在交互和场景上向「听歌」靠拢，方便用户把下载的 MV、赞美诗、课程等**当音频来听**。技术栈为原生 Android（Java/Kotlin），纯本地、无服务端，优先使用现成播放引擎与组件。

## Core Vision

### Problem Statement

不少用户会从 YouTube 等渠道下载 MV、赞美诗、演讲等视频到本地，希望像听歌一样使用：锁屏播放、列表循环、通知栏切歌。但现有方案难以同时满足：**音乐 APP** 主要面向在线/本地音频（如 mp3），不把本地视频当「歌」来管理；**通用视频播放器** 更强调「看」，息屏、后台、通知栏体验往往不如音乐 APP，且缺少「听歌列表」「按歌单播放」等习惯。因此出现**「有视频却无法当歌听」**的缺口。

### Problem Impact

最受影响的场景包括：通勤、步行、骑车、运动时想听赞美诗、课程、播客类视频。若问题不解决，用户要么只能开着屏幕耗电播放，要么被迫先把视频转成音频再导入音乐 APP，流程繁琐，也占额外存储。

### Why Existing Solutions Fall Short

- 音乐 APP：以音频格式和在线曲库为主，本地视频导入与「听歌」体验未打通。
- 通用视频播放器：侧重播放与看，锁屏/后台策略、通知栏样式、播放列表心智大多不像「听歌」，难以形成「当歌听」的完整体验。
- 手动转码再导入音乐 APP：步骤多、耗时间，且不利于「一个文件既想看又能听」的用法。

### Proposed Solution

做一款**本地视频播放器**，重点支持「**锁屏听视频里的音频**」：

- **本质是视频播放器**：扫描、浏览、管理本地视频，支持卡片式浏览与多选添加。
- **听歌式用法**：用户自建播放列表/歌单并命名（如「赞美诗」「通勤听」），提供播放/暂停、上一首/下一首、单曲/列表循环、随机播放；支持**息屏播放**与**酷狗/QQ 音乐风格的通知栏**控制。
- **核心诉求**：**主要为了听视频的音频**——不是再造一个「音乐播放器」，而是让现有视频文件能像歌一样在锁屏、后台、通知栏里听。
- **看与听**：既可正常看视频，也可切到「当歌听」模式（锁屏/后台播音频）；并支持收藏、多标签、按标签/歌单/收藏/历史查找。
- **技术栈**：原生 Android（Java/Kotlin）、纯本地存储与数据库，优先采用现成播放引擎与组件，必要时再自研补强。

### Key Differentiators

- **定位清晰**：就是**本地视频播放器**，差异在于**可以锁屏听、像歌曲播放器那样播视频来听**，本质是「把视频当音源用」。
- **纯本地、零服务端**：无账号、无云端，适合注重隐私与离线的用户，也便于个人开发者维护。
- **技术策略**：组件/现成方案优先，数据库与架构预留日后对接服务端的可能，便于从纯本地平滑演进。

## Target Users

### Primary Users

**日常用户（锁屏听）**  
- **典型场景**：走路、上下班、骑车时听赞美诗、课程、播客类视频里的音频。  
- **核心诉求**：锁屏听、顺序/列表循环、通知栏控制；少操作、少学习成本。  
- **成功体验**：选好歌单即可锁屏听，不必开屏、不占视觉。

**重度用户（多列表、多分类）**  
- **典型场景**：本地视频多，需要按歌单、标签、收藏、历史快速找到要听/看的内容。  
- **核心诉求**：自建多个歌单、多标签、拖拽排序、批量操作、按标题搜索。  
- **成功体验**：大量视频仍能快速整理、快速切到想听的那一类。

### Secondary Users

- **当前**：无。产品为纯本地、个人使用，不做分享与多人协同。  
- **后期可选**：若有需要，可再考虑「家人共享 / 设备内多 profile / 分享歌单」等能力，均作为可选扩展。

### User Journey

- **发现**：用户已有本地视频（如从 YouTube 等下载），希望当歌听，主动寻找「能锁屏听的本地视频播放器」。  
- **首次使用**：安装后扫描本地视频 → 建第一个歌单、勾选要听的视频 → 播放，锁屏试听。  
- **日常使用**：打开 APP 选歌单 → 播放 → 锁屏/切到其他 App，用通知栏控制；或需要时再进 APP 看视频。  
- **价值感知**：「不用转格式、不用开屏，和听歌一样听视频。」  
- **长期**：歌单/标签越建越多，按场景（通勤、运动、学习）分类，形成稳定使用习惯。

## Success Metrics

### User Success

- **核心结果**：用户能在锁屏/后台状态下，用歌单连续听本地视频里的音频，且用通知栏完成播放/暂停/切歌，无需开屏。
- **可观测行为**：用户会建至少一个歌单、往其中添加本地视频，并在一周内多次使用「锁屏听」或「通知栏控制」。
- **价值时刻**：用户体会到「不用转格式、不用开屏，就像听歌一样听视频」，并愿意在通勤、运动等场景下持续使用。

### Business Objectives / 项目目标

- **当前**：个人项目，无营收、无用户增长 KPI。目标为「做出可用的本地视频播放器，支持锁屏听、歌单、基础管理」。
- **可选后续**：若日后考虑分发或轻量商业化，再补充下载量、留存、满意度等；本次产品简介中**不作为必填**，仅作「后期可选」说明。

## MVP Scope

### Core Features

- **本地视频扫描与添加**：扫描本地视频、卡片式展示、多选添加；支持拖拽即添加、按添加时间排序。
- **播放列表/歌单**：用户自建歌单并命名；支持先建空列表再批量添加、拖拽排序、批量操作。
- **播放控制与息屏**：播放/暂停、上一首/下一首、单曲/列表/随机循环；**息屏播放**；**酷狗/QQ 音乐风格通知栏**控制。
- **本地存储**：Room 存播放列表、收藏、标签、历史；数据库设计预留日后服务端对接。
- **收藏与分类**：收藏（与喜欢合并）；多标签；列表/标签视图切换；按标签、歌单、收藏、历史查找。
- **搜索**：按视频标题模糊搜索、历史搜索、搜索高亮。
- **看与听**：正常看视频；倍速、画中画、睡眠定时；播放列表封面。
- **歌词（MVP 内）**：本地歌词文件、保存路径、富文本展示（图+文）。
- **基础信息与交互**：视频信息（时长、分辨率、文件大小）；卡片式浏览、悬浮控制；手势（滑动切歌、双击暂停等）；最近播放、继续观看、最近添加。

### Out of Scope for MVP

- 新用户引导步骤
- 场景模式、智能推荐（依赖服务端）
- 音频转文字歌词、歌词滚动同步
- 音效设置（均衡器、3D 环绕等）
- 播放速度记忆、章节、离线下载（若有服务端再考虑）
- 分享、家人共享、多设备同步（后期可选）

### MVP Success Criteria

- 路径「扫描本地视频 → 建歌单 → 添加视频 → 锁屏播放 → 通知栏控制」可完整走通。
- 本人能在一周内多次用该 APP 锁屏听视频，并愿意继续使用。

### Future Vision

- **后期可选**：新用户引导、音频转文字歌词、歌词滚动、音效、播放速度记忆、章节、离线下载（若有服务端）。
- **更远期**：分享、家人共享、多 profile、分发或轻量商业化；架构已预留从纯本地平滑演进的可能。
